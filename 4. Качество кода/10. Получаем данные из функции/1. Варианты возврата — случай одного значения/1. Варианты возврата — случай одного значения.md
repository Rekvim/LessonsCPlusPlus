## Варианты возврата — случай одного значения

Назначение любой функции — это получение информации, вычисление или выполнение действия, имеющего побочный эффект. Есть несколько способов получить из функции данные — результат вычисления или отчёт о выполнении действия:

- возвращение через `return`;
- возвращение через параметр;
- исключение.

Из них самый явный, наглядный и понятный — `return`. Он позволяет выполнить волшебные оптимизации, недоступные для других способов — Copy Elision и NRVO

#### По ссылке или по значению

Вам, вероятно, уже надоел этот вопрос после первой темы. Тогда вас обрадует, что сейчас мы разберёмся с ним раз и навсегда.

Для каких случаев подходит возврат по ссылке? — Получение информации.

Возвращение по ссылке не передаёт владение. Если вернуть ссылку на объект, возникший в процессе работы функции и уничтожившийся после её завершения, возникнет неприятная ситуация — ошибка, которую компилятор не всегда сможет выявить.

Метод может возвращать объект, которым владеет класс. Это типичная ситуация. Чтобы выбрать, как возвращать — по ссылке или по значению, — нужно задать несколько вопросов.

- Предоставляем ли мы возможность менять объект после получения? Тогда нужен возврат по неконстантной ссылке.
- Объект тяжелый или его копирование имеет побочный эффект? Тогда нужен возврат по константной ссылке, NRVO не сработает при возвращении поля класса.
- Объект лёгкий и просто копируется? — Тогда можно возвращать по значению.

Выберите правильный метод возврата для каждой функции, не передающей владение:
```cpp
std::vector<int>::operator[](size_t pos)
```
— По ссылке — предоставляем возможность менять.
```cpp
std::vector<int>::size()
```
— По значению — лёгкое копирование.
```cpp
// когорта
class Cohort {
    // ...
public:
    // метод возвращает вектор результатов студента по каждой задаче
    <???> GetStudentResults(std::string_view student_name) const {
        return students_results_.at(student_name);
    }

private:
    std::map<std::string_view, std::vector<TaskResult>> students_results_;
}; 
```
— По константной ссылке — затратное копирование

#### Возврат значения или пустоты

Бывает, что функции нечего возвратить. Как в сказке о золотой рыбке, когда старик первый раз с пустыми руками возвращается к старухе. Но отругать функцию и послать её снова за значением нельзя, вместо этого нужен способ хотя бы определить, вернула функция что-нибудь или нет.

Например, мы пишем метод подсчёта среднего возраста студентов:
```cpp
class Cohort {
    // ...
public:
    double ComputeMeanAge() const {
        int total_sum = 0;
        size_t total_count = students.size();
        for (const auto& student : students) {
            total_sum += student.GetAge();
        }
        
        return static_cast<double>(total_sum) / total_count;
    }
};
```
В этой программе есть ошибка: если студентов нет, то нет среднего арифметического. Функция должна сообщить, что оно не определено, а вместо этого она делит на ноль и роняет программу. Есть три варианта решения этой проблемы:

1. Сказать, что пользователь сам виноват, проверить корректность данных перед вызовом — его ответственность. Это можно оправдать желанием сэкономить на проверке. В определённых случаях, где производительность критически важна, можно прибегать к такому аргументу.

2. Выбросить исключение. Этот способ будет разбираться в одном из последующих уроков.

3. Попробуйте догадаться сами.

Какой способ лучше и универсальнее для возврата значения, которое при определённых условиях может отсутствовать? — Вернуть `optional`.


Класс `optional` отлично подходит для такого случая. Он наглядно покажет по сигнатуре, что функция может возвратиться «с пустыми руками». Перепишем метод `ComputeMeanAge`, исправив эту ошибку:
```cpp
class Cohort {
    // ...
public:
    std::optional<double> ComputeMeanAge() const {
        int total_sum = 0;
        size_t total_count = students.size();

        // обработаем случай отсутствия студентов
        if (total_count == 0) {
            return std::nullopt;
        }

        for (const auto& student : students) {
            total_sum += student.GetAge();
        }
        
        return static_cast<double>(total_sum) / total_count;
    }
};
```
#### Возврат с несколькими альтернативами
Представим, что задача — не просто сказать, что значения нет, но и сообщить подробности об ошибке. Например, мы пишем метод, выдающий студенту диплом — да, да, вам осталось не так много до этого торжественного момента:
```cpp
class Diploma {
    // ...
};

class Cohort {
public:
    Diploma Graduate(int student_id);
    // ...
};
```
Однако метод `Graduate` должен проверить, что студент уже выполнил все задания и готов к получению диплома. В противном случае этот метод возвратит ошибку, которую можно представить значением `enum`:
```cpp
enum class GraduateFailReason {
    NOT_ALL_SPRINTS_PASSED,    // не все спринты пройдены
    FINAL_TASK_NOT_APPROVED,   // финальное задание не прошло ревью
    INCOMPLETE_FUNDING,        // неполная оплата
    DIPLOMA_ALREADY_RECEIVED   // диплом уже получен
};
```
Возвращаемое значение имеет две альтернативы — сам диплом или код ошибки.

Какой способ лучше для этого случая? — Вернуть `variant`.

Специально для представления нескольких альтернатив в стандартной библиотеке есть класс `variant`. Он вполне подходит для представления возвращаемого значения. С ним сигнатура становится внятной:
```cpp
std::variant<Diploma, GraduateFailReason> Graduate(int student_id);
```
В этом уроке затронут случай, когда из функции или метода нужно получить максимум одно значение. Далее разберём возврат нескольких объектов, набора и другие способы возврата. Для этого проведём несколько бенчмарков.
