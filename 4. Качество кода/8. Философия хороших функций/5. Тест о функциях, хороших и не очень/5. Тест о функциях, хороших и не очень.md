## Тест о функциях, хороших и не очень

Впереди вас ждут несколько вопросов о свойствах функций. Код писать не нужно — вы будете оценивать чужой. Это важный навык, который позволяет критически подходить к собственному коду.

Вам предстоит разобраться, что делает функция, исходя из её сигнатуры. По сигнатуре можно понять следующее:

- Что функция обещает делать? Обычно об этом говорят названия функции и её аргументов.

- Что функции позволено делать? Здесь речь скорее о типах аргументов и возвращаемом значении.

Вопросы о трактовках, совместимых с сигнатурой, относятся к тому, что функции теоретически позволено делать. Ведь в отношениях с функциями часть обещать — не значит жениться.

Посмотрите примеры, где функциональность можно полностью покрыть юнит-тестами, а где нет. Если одного лишь кода функции недостаточно для однозначного ответа, исходите из худшего случая.

Нельзя
```cpp
size_t ComputeWordCount() {
    size_t word_count = 0;
    string word;
    while (сin >> word) {
        ++word_count;
    }
    return word_count;
} 
```
На эту функцию невозможно написать юнит-тесты, потому что данные берутся напрямую из стандартного потока ввода. Иными словами, функция плоха тем, что работает с глобальной переменной `cin`, влиять на которую в юнит-тестах возможности нет.

Нельзя
```cpp
size_t ComputeWordCount(string_view s) {
    size_t word_count = 0;
    for (size_t i = 1; i < s.length(); ++i) {
        word_count += s[i] == ' ' && s[i - 1] != ' ';
    }
    return word_count;
}
```
Действительно, у этой функции один вход — строка `s` — и один выход — количество слов. В юнит-тестах достаточно вызывать эту функцию от некоторой строки и сравнивать полученное число с ожидаемым. Кстати, видите ли вы ошибку в этой реализации?

Можно
```cpp
const size_t MAX_STRING_LENGTH = 100;

size_t ComputeWordCount(string_view s) {
    size_t word_count = 0;
    for (size_t i = 1; i < min(s.length(), MAX_STRING_LENGTH); ++i) {
        word_count += s[i] == ' ' && s[i - 1] != ' ';
    }
    return word_count;
}
```
Действительно, использование глобальной константы не мешает писать юнит-тесты на функцию. Этому есть несколько объяснений:

- С тем же успехом в функции могло быть просто использовано число 100, без выселения его в именованную константу;

- Одно из ключевых требований к тестам — воспроизводимость. Константность глобальной переменной исключает влияние других функций на её значение, поэтому и в тестах, и в реальном окружении эта функция будет работать одинаково.

Можно
```cpp
size_t ComputeWordCount(istream& stream) {
    size_t word_count = 0;
    string word;
    while (stream >> word) {
        ++word_count;
    }
    return word_count;
}
```
Да, эту функцию можно протестировать, передавая в неё `stringstream`.

Нельзя
```cpp
size_t ComputeWordCount() {
    size_t word_count = 0;
    for (size_t i = 1; i < current_line.length(); ++i) {
        word_count += current_line[i] == ' ' && current_line[i - 1] != ' ';
    }
    return word_count;
}
```
Эта функция берёт слово для обработки из глобальной переменной `current_line`. Если повезёт, переменная будет объявлена в заголовочном файле, и в коде юнит-тестов можно будет влиять на неё произвольным образом. Но наиболее вероятно, что эта переменная будет объявлена рядом с реализацией и сможет изменяться только вызовом других функций. Их вполне может оказаться недостаточно, чтобы получить в переменной `current_line` любое нужное в тестах значение.

Дана сигнатура функции. Отметьте те трактовки её функциональности, которые совместимы с такой сигнатурой. Считайте, что функция не позволяет себе «грязные» и опасные приёмы вроде изменения константного объекта с использованием `const_cast`.

```cpp
int ComputeGoodNumberCount(const vector<int>& numbers); 
```

Правильный ответ
Функция перебирает все числа входного вектора и для каждого проверяет, хорошее ли оно. Количество хороших чисел возвращает через `return`. Дополнительно выводит в стандартный поток вывода список хороших чисел.
Действительно, даже за безобидной сигнатурой может скрываться взаимодействие со стандартными потоками ввода и вывода. Но такое может случиться с любой функцией, это не повод для паранойи. Поскольку подобное поведение не ожидается и не тестируется, сигнатуру следует пересмотреть.

Правильный ответ
Функция перебирает все числа входного вектора и для каждого проверяет, хорошее ли оно. Поскольку проверка на хорошесть — тяжёлая операция, для её ускорения используется глобальный кэш, представляющий собой неконстантный словарь. Количество хороших чисел возвращается через `return`.
Такая реализация функции возможна. Поскольку это поведение не ожидается и не тестируется, такой кэш стоит сделать параметром функции или полем класса, методом которого будет функция.

Функция не может менять вектор, потому что он константный.

Действительно, это поведение функции не только формально соответствует сигнатуре, но и наиболее ожидаемо. Во многом благодаря названию функции.

Неправильный ответ
Функция перебирает все числа входного вектора, для каждого проверяет, хорошее ли оно, и зануляет все числа, которые хорошими не являются. Количество хороших чисел возвращает через `return`.
Функция не может менять содержимое вектора, потому что он константный.

Дана сигнатура функции. Отметьте те трактовки её функциональности, которые допустимы при этой сигнатуре. Значение слова `Factor` в ней однозначно не определено.

```cpp
template <typename InputIt>
int ComputeFactor(InputIt begin, InputIt end); 
```

Правильный ответ
Функция сортирует переданный диапазон и возвращает число, оказавшееся восьмым или последним, если восьмого числа нет.
Вполне возможно, что под словом “factor” автор функции имел в виду восьмое по величине число из набора. При этом название `InputIt` не гарантирует, что элементы не будут изменены. Это не повод страдать при её использовании, но желающие перестраховаться могут передавать в неё исключительно константные итераторы.

Неправильный ответ
Функция принимает два итератора — на вектор и на множество. Последовательно получает из них 100 пар чисел и возвращает количество тех пар, сумма чисел в которых хорошая.
Сигнатура функции требует, чтобы итераторы имели одинаковый тип.

Правильный ответ
Функция перебирает все числа входного набора и для каждого проверяет, хорошее ли оно. Количество хороших чисел возвращает через `return`.
Это вполне возможно, но не угадывается по сигнатуре из-за плохих названий функции и параметров.

Правильный ответ

Это вполне возможно, но не угадывается по сигнатуре из-за плохих названий функции и параметров. Ничто даже не даёт понять, что функция принимает именно строки.

Дана сигнатура функции. Отметьте те трактовки её функциональности, которые следуют из сигнатуры.
```cpp
void FindGoodNumbers(const vector<int>& numbers, vector<int>& good_numbers); 
```

Правильный ответ
Функция перебирает все числа вектора `numbers` и для каждого проверяет, хорошее ли оно. Все хорошие числа записываются в вектор `good_numbers`, который предварительно очищается и благодаря этому в итоге содержит все хорошие числа вектора `numbers`.
Действительно, это наиболее вероятный вариант поведения данной функции. Но тогда было бы логичнее вернуть `good_numbers` через `return`.

Правильный ответ
Функция перебирает все числа вектора numbers и для каждого проверяет, хорошее ли оно. Все хорошие числа добавляются в конец вектора `good_numbers`.
Такая реализация не только формально соответствует сигнатуре, но и вполне возможна, если в точке вызова этой функции стоит задача собрать хорошие числа из разных мест. Проблема в том, что такая трактовка легко путается с версией, где `good_numbers` предварительно очищается. Можно было бы посетовать на название и предложить в этой версии заменить `Find` на `Add` или `Update`, но придумывать однозначные и в то же время короткие названия — сложное искусство. Проще всё же научиться использовать прозрачные сигнатуры и для версии без очистки `good_numbers` оставить эту, а в версии с очисткой использовать `return`.

Правильный ответ
Функция перебирает все положительные числа вектора `numbers` и проверяет каждое из них на хорошесть путём поиска в `good_numbers`. Если очередное число нашлось в `good_numbers`, соответствующий элемент `good_numbers` умножается на −1. Таким образом, использованные числа помечаются и не могут быть использованы дважды.
Такую реализацию тоже можно себе представить, но её проблема в том, что она маскирует за простотой сигнатуры сложность логики: числа хитро помечаются и не переиспользуются. Хорошим решением было бы использовать для `good_numbers` специальный класс.
